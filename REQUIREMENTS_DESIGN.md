## 一、项目概述

### 1.1 定位
面向**公众·企业·政府·监管·执法·行政**六位一体的环保智慧执法平台，以开源知识库为基座，AI智能体为核心，实现从线索发现到结案归档的全链路智能闭环。

### 1.2 核心价值
- **知识驱动**：开源知识图谱+多源采集，形成环保执法领域最完善知识基座
- **AI闭环**：EPB-Agent智能体贯穿感知→推理→执行→反馈全链路
- **六位一体**：公众举报→企业合规→政府决策→监管预警→执法闭环→行政支撑
- **开源开放**：MIT协议开源，社区共建，API开放

### 1.3 目标用户与场景

| 角色 | 核心场景 | 闭环路径 |
|------|---------|---------|
| 👥 公众 | 举报/查询/监督/评价 | 举报→受理→核实→处置→公示→评价 |
| 🏢 企业 | 合规自查/监测/整改/培训 | 许可→监测→自查→整改→达标→信用 |
| 🏛 政府 | 态势感知/决策/政策/汇报 | 感知→分析→决策→执行→评估 |
| 🔍 监管 | 在线监测/预警/审查/溯源 | 监测→AI识别→预警→执法→闭环 |
| 🚔 执法 | 现场检查/证据/文书/结案 | 线索→立案→检查→证据→审查→处罚→执行→结案 |
| 🏢 行政 | 科室职能/许可/监测/宣传 | 科室工作门户·日常行政支撑 |

## 三、EPB-Agent 智能体架构

### 3.1 多层架构
```
┌─────────────────────────────────────────────────────────────────┐
│                    🧠 EPB-Agent 智能体                         │
├─────────────────────────────────────────────────────────────────┤
│  🎬 执行层  文书生成·视频生成·预警推送·任务分派              │
├─────────────────────────────────────────────────────────────────┤
│  🧠 推理层  LLM推理·规则引擎·法条匹配·裁量计算              │
│            ├─ 本地: Qwen2.5(1.5B) ← 简单/离线/隐私          │
│            ├─ 云端: Claude/GPT-4o ← 复杂推理/长文本          │
│            └─ 视觉: GPT-4oV/YOLOv8 ← 场景识别/视频AI        │
├─────────────────────────────────────────────────────────────────┤
│  👁 感知层  摄像头·无人机·无人船·传感器·举报·用电监控        │
├─────────────────────────────────────────────────────────────────┤
│  📚 知识层  法规库·案例库·标准库·行业知识·专家经验            │
└─────────────────────────────────────────────────────────────────┘
🔄 闭环机制: 感知→推理→执行→反馈→学习→感知(持续迭代)
📊 决策路径: 数据→信息→知识→智慧→行动
```

## 四、功能需求（8大模块×5角色）

### 模块1：开源环保知识库 📚
数据来源×8 + 采集进度×6类 + 条目列表（编号/类型/标题/来源/状态/日期）+ 社区投稿 + KB分类筛选

执法闭环（线索→立案→检查→证据→审查→处罚→执行→结案，全8步规范支撑）+ 现场终端 + 非现场执法

8科室科室工作门户（见第二章）

### 模块4：AI智能中心 🧠（新增）
EPB-Agent架构 + 大模型清单对比 + 自研算法矩阵

### 模块5：工作台 📋
日程+任务+日志+企业查询+AI汇报

### 模块6：培训学院 🎓
场景化实训+AI模拟询问+能力认证

### 模块7：物联感知 📡
5大感知源+三级预警+设备管理

### 模块8：成果展示 🎬
