## 零、平台定位与部署模式

### 0.1 通用智能体基座

平台本身是一个**通用环保智能体基座**，不绑定任何特定区域。内置全国通用的环保法规（28部）、执法案例（101条）、违法类型映射（15类）、知识图谱（8大领域）作为基础知识库。

### 0.2 本地化部署模式

给某个单位部署时，只需注入该单位的区域配置和数据，即可快速形成本地化服务：

```
平台基座（21页面 + 28API + 6角色 + 101案例 + 28法规）
  + 租户配置（单位名称/类型/区域）
  + 本地法规（本地地方法规）
  + 本地案例（本地执法案例）
  + 辖区企业名录
  + 监测站点配置
  + 组织架构和用户
  = 本单位环保智能体平台（2-4小时完成部署）
```

### 0.3 三级部署体系

支持**省→市→区县**三级独立部署，数据逐级汇聚：

| 层级 | 部署单位 | 数据范围 | 核心功能 |
|------|---------|---------|---------|
| 省级 | 生态环境厅 | 全省汇总 | 全省态势感知、跨市协调、省级政策下发 |
| 市级 | 生态环境局 | 全市汇总 | 全市执法调度、区县考核、市级任务下达 |
| 区县 | 生态环境分局 | 本区县 | 日常执法检查、企业监管、举报受理 |
| 企业 | 企业集团/园区 | 本企业/园区 | EHS管理、自查整改、设备运维 |

**数据流向**：区县采集 → 市局汇总 → 省厅感知（逐级汇聚，不可逆向查看）

### 0.4 部署初始化流程

```
1. 安装平台基座（21页面+28API+6角色体系）
2. 填写租户配置（单位名称/类型/区域）→ tenant.json
3. 导入本地法规（地方法规加入 law_index.json）
4. 导入本地案例（本地执法案例加入 cases.json）
5. 导入辖区企业名录 → enterprises.json
6. 配置监测站点 → stations.json
7. 配置组织架构和用户 → orgs.json + users.json
8. 启动服务，各角色登录即看到本地化工作台
```

平台启动后还支持自动采集补充：爬虫定时采集省市生态环境部门最新法规和案例、AI联网搜索补充最新政策、在线监测设备自动注册接入、企业自行注册填报。

## 一、平台核心架构与设计思想

### 1.1 设计思想

**一句话定位**：环保行业最专业的智能体平台，让政府、企业、第三方、群众四方角色通过统一平台协同工作，AI 驱动全流程效率提升。

**三个核心设计原则**：

1. **角色驱动（Role-Driven）** — 不同角色看到不同的工作台、不同的模块、不同的数据。执法人员看到执法终端和证据链，企业人员看到自查和整改，公众看到举报和案例。6 种角色，6 种工作台。

2. **闭环协同（Closed-Loop）** — 所有业务流程都是闭环的：线索→立案→检查→证据→审查→处罚→执行→结案。政府下达任务→企业整改→第三方核查→群众监督→归档反馈。不产生孤立数据。

3. **AI 赋能（AI-Empowered）** — 不是简单的信息管理系统，而是每个环节都有 AI 辅助：智能分析案情匹配法条、场景识别自动建议证据采集、文书 AI 自动生成、预警 AI 融合多源数据。

### 1.2 四层架构

```
┌─────────────────────────────────────────────────────────────┐
│                     🎬 执行层（前端）                        │
│  PC端14页面 + 移动端4页面 + 登录引导 + 数据驾驶舱 + EHS管理  │
│  6种角色定制工作台，权限守卫自动控制访问                     │
├─────────────────────────────────────────────────────────────┤
│                     🧠 推理层（AI引擎）                      │
│  LawMatch法条匹配 | SceneSense场景识别 | ViolationAssess评估 │
│  DocGen文书生成 | FusionAlert融合预警 | VideoNarrative视频   │
├─────────────────────────────────────────────────────────────┤
│                     👁 感知层（设备接入）                    │
│  智能眼镜 | 执法记录仪 | 便携传感器 | 在线监测 | 用电监控    │
│  视频监控 | 无人机 | 蓝牙设备 | 微信小程序                   │
├─────────────────────────────────────────────────────────────┤
│                     📚 知识层（数据底座）                    │
│  101条案例 | 28部法规 | 15类违法映射 | 8大知识图谱           │
│  用户体系 | 任务体系 | 举报体系 | 设备体系                   │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 技术栈

| 层级 | 当前技术 | 演进方向 |
|------|---------|---------|
| 前端 | 原生 HTML/CSS/JS（21页面） | Vue3 + Arco Design（当维护成本过高时迁移） |
| 后端 | Python http.server（28 API） | FastAPI（异步+自动文档） |
| 数据 | JSON 文件存储 | SQLite + Redis（结构化+缓存） |
| AI | 规则引擎 + 关键词匹配 | 大模型 + 向量检索（Milvus/Faiss） |
| 设备 | 无直连 | MQTT + WebSocket + 蓝牙 5.0 |
| 部署 | 本地单机 | Docker + Nginx 反向代理 |

## 三、产品完善规划 — 执法终端设备接入

### 3.1 智能眼镜集成方案

**行业参考**：2026年北京经开区率先推出 AI+AR 智能执法眼镜，实现"看到即查到"——全过程自动识别、实时核验、闭环留档。

**支持的智能眼镜型号**：
| 型号 | 通信方式 | 核心能力 | 适用场景 |
|------|---------|---------|---------|
| RealWear HMT-1 | WiFi/蓝牙 | 第一视角拍摄、语音交互、AR提示 | 现场执法全程记录 |
| Google Glass EE | WiFi/蓝牙 | AR显示、拍照录像、语音控制 | 督查核查 |
| Rokid X-Craft | WiFi/5G | 防爆设计、AR叠加、AI识别 | 化工园区执法 |
| EPSON Moverio | WiFi/蓝牙 | 双目AR显示、高透光率 | 设备巡检 |

**眼镜接入流程**：
```
执法人员佩戴智能眼镜
  ↓ 眼镜通过WiFi连接到平台
  ↓ 平台下发督查清单/执法任务到眼镜AR显示
  ↓ 眼镜第一视角实时回传视频流
  ↓ AI实时分析画面（SceneSense算法）
  ↓ AR叠加提示：「检测到疑似废气超标，请采集以下证据...」
  ↓ 执法人员按提示拍照/录像，眼镜自动标注时间和GPS
  ↓ 证据自动上传到案件证据链
  ↓ AI匹配法条，生成执法文书草稿
```

### 3.2 执法记录仪集成

**支持的设备**：
- 海康/大华 4G版执法记录仪（全程录像录音、GPS定位、实时回传）
- 蓝牙便携式拾音器（远程指挥中心实时听取现场对话）

**集成能力**：
- 实时视频回传到指挥中心
- GPS轨迹记录和回放
- 一键报警触发应急响应
- 录像自动关联案件编号
- 设备电量/在线状态监控

### 3.3 便携传感器集成

| 传感器类型 | 检测指标 | 连接方式 | 数据频率 |
|-----------|---------|---------|---------|
| 便携气体检测仪 | VOCs/SO₂/NOx/CO/O₃ | 蓝牙5.0 | 1次/秒 |
| 便携水质检测仪 | pH/COD/氨氮/总磷 | 蓝牙5.0 | 1次/10秒 |
| 噪声检测仪 | dB(A) | 蓝牙5.0 | 1次/秒 |
| 粉尘检测仪 | PM2.5/PM10/TSP | 蓝牙5.0 | 1次/秒 |
| 红外热成像仪 | 温度分布 | WiFi | 1次/5秒 |

**数据流**：
```
传感器采集 → 蓝牙传输到手机/平板 → MQTT上传到平台
  → 实时展示在执法终端 → 超标自动触发预警
  → 数据关联案件证据链 → 生成监测报告
```

## 四、非现场监管执法方案

### 4.1 非现场执法闭环

```
┌─ 数据采集 ─┐    ┌─ AI分析 ───┐    ┌─ 证据固定 ─┐    ┌─ 执法处置 ─┐
│ 在线监测   │    │ 超标识别   │    │ 数据截图   │    │ 远程询问   │
│ 用电监控   │ →  │ 异常检测   │ →  │ 时间戳     │ →  │ 文书送达   │
│ 视频监控   │    │ 趋势分析   │    │ 数字签名   │    │ 整改跟踪   │
│ 举报线索   │    │ 风险评分   │    │ 区块链存证 │    │ 结果公示   │
└────────────┘    └────────────┘    └────────────┘    └────────────┘
```

### 4.2 FusionAlert 多源融合预警算法

**输入数据源**：
1. 在线监测数据（COD/氨氮/SO₂/NOx/烟尘等）
2. 用电监控数据（生产用电/治污用电/异常停运）
3. 视频监控数据（AI识别烟尘/污水/扬尘/人员活动）
4. 举报数据（举报频率/类型/区域）
5. 历史执法数据（违法记录/处罚记录/信用等级）

**风险评分模型**：
```
风险分 = Σ(数据源权重 × 异常程度)
  在线监测异常 × 0.35
  用电异常 × 0.20
  视频异常 × 0.20
  举报频率 × 0.10
  历史记录 × 0.15
```

**预警分级**：
| 分数 | 等级 | 颜色 | 响应措施 |
|------|------|------|---------|
| ≥80 | 高风险 | 红色 | 立即派执法组现场检查 |
| 60-79 | 中风险 | 橙色 | 远程约谈+限期自查 |
| 40-59 | 低风险 | 黄色 | 加强在线监控频率 |
| <40 | 正常 | 绿色 | 常规监管 |

## 七、数据安全与合规

### 7.1 电子证据法律效力

| 要求 | 实现方式 |
|------|---------|
| 时间戳 | NTP服务器同步 + 可信时间戳服务 |
| 数字签名 | RSA/SM2数字签名，防篡改 |
| 数据完整性 | SHA-256哈希校验 |
| 不可抵赖 | 区块链存证（可选 Hyperledger Fabric） |
| 全程记录 | 执法全过程录像录音，符合《行政处罚法》 |

### 7.2 数据保护

| 要求 | 实现方式 |
|------|---------|
| 传输安全 | TLS 1.3 全链路加密 |
| 存储安全 | AES-256 加密存储敏感数据 |
| 访问控制 | RBAC 角色权限（6种角色42个权限点） |
| 数据脱敏 | 举报人信息脱敏存储，手机号中间4位隐藏 |
| 审计日志 | 所有操作可追溯，不可删除 |
| 数据保留 | 案件档案 ≥ 10年 |

### 7.3 离线能力

| 场景 | 方案 |
|------|------|
| 执法现场无网络 | 本地缓存 + 离线AI（Qwen2.5:1.5B） |
| 语音提醒离线 | 预置提醒模板 + 本地TTS |
| 证据数据离线暂存 | 本地SQLite + 网络恢复后自动同步 |
| 离线文书生成 | 本地模板 + 填充式生成 |

## 九、总结

环保智慧执法平台已建成 21 个页面、28 个 API、6 种角色体系的核心框架，覆盖了执法终端、AI 分析、知识库、物联感知、协同工作、举报管理、培训学院、数据驾驶舱、企业 EHS 等全链路功能。
