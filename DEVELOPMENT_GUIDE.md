## 一、系统愿景与定位

### 1.1 愿景
打造环保行业最专业的智能体（Skills）平台，服务全社会环保职能体系。

服务对象覆盖四类主体，形成全链路闭环：
1. **政府端**（所有环保相关单位）— 执法局、监测中心、督查组、第三方检查机构，通过接口各自开展工作
2. **企业端** — 自查自纠、日常监测、合规管理
3. **群众端** — 监督举报、结果查询、评价反馈
4. **第三方机构** — 检查督查、技术支撑、数据服务

最终目标：成长为环保行业最专业的 Skills，任何环保相关单位和个人都能通过平台接口开展工作。

### 1.2 核心能力
1. **环保知识库** — 法规、案例、标准、行业知识，全行业共享
2. **现场执法终端** — 穿戴设备捕捉环境情况，语音提醒采集证据，形成证据链
3. **非现场执法** — 远程监控、数据分析、在线预警
4. **第三方监管督查** — 穿戴设备辅助监管人员完成督查工作，生成督办清单
5. **群众举报** — 多渠道接入、智能分发、闭环处理
6. **企业自查** — 自检自纠、日常监测、合规培训
7. **协同工作台** — 政府下达任务→企业整改→第三方核查→群众监督，四方合力
8. **开放接口层** — 标准化 API，任何环保相关单位均可接入

## 三、架构设计 — 智能体闭环

### 3.1 EPB-Agent 四层架构
```
┌─────────────────────────────────────────────────────────┐
│                    🧠 EPB-Agent 智能体                   │
├─────────────────────────────────────────────────────────┤
│  🎬 执行层  文书生成·语音提醒·预警推送·任务分派        │
├─────────────────────────────────────────────────────────┤
│  🧠 推理层  LLM推理·规则引擎·法条匹配·裁量计算         │
│            ├─ 本地: Qwen2.5(1.5B) ← 简单/离线/隐私     │
│            ├─ 云端: Claude/GPT-4o ← 复杂推理/长文本     │
│            └─ 视觉: GPT-4oV/YOLOv8 ← 场景识别/视频AI   │
├─────────────────────────────────────────────────────────┤
│  👁 感知层  穿戴设备·摄像头·传感器·举报·用电监控        │

└─────────────────────────────────────────────────────────┘
🔄 闭环: 感知→推理→执行→反馈→学习→感知(持续迭代)
```

### 3.2 模块详细设计

#### 3.2.1 环保知识库引擎
- **法规库**：环保法律、行政法规、地方法规、行业标准、排放标准
- **案例库**：典型违法案例、处罚案例、判例分析
- **知识图谱**：8大分类（水/大气/土壤/固废/噪声/辐射/生态/环评）
- **采集系统**：8大来源自动采集 + 社区投稿 + 人工录入
- **智能检索**：关键词→语义扩展→向量检索→排序→校验（LawMatch 算法）

#### 3.2.2 现场执法终端（穿戴设备集成）
**核心流程**：
```
执法人员到达现场
  ↓
穿戴设备启动（摄像头/传感器/麦克风）
  ↓
设备捕捉环境情况 → AI 实时分析
  ↓
语音提醒：「检测到疑似废气超标排放，请采集以下证据：
  1. 拍摄排放口照片
  2. 采集废气样本
  3. 记录企业生产状态
  4. 查阅排污许可证」
  ↓
执法人员按提示采集 → 形成完整证据链
  ↓
AI 自动匹配法条 → 生成执法文书草稿
  ↓
上传至系统 → 进入案件流程
```

**穿戴设备选型建议**：
| 设备类型 | 推荐方案 | 用途 |
|---------|---------|------|
| 智能头盔 | RealWear HMT-1 / Google Glass Enterprise | 第一视角拍摄、语音交互、AR提示 |
| 执法记录仪 | 海康/大华 4G版 | 全程录像录音、GPS定位、实时回传 |
| 便携传感器 | 便携式气体检测仪（蓝牙） | 实时检测 VOCs/SO2/NOx/PM2.5 |
| 智能手表 | Apple Watch / 华为手表 | 振动提醒、快速操作、健康监测 |

**语音提醒系统设计**：
- ASR（语音识别）：Whisper-Large / 讯飞 ASR
- NLU（理解）：LLM 分析现场情况 → 匹配知识库 → 生成提醒
- TTS（语音合成）：CosyVoice / 讯飞 TTS
- 离线方案：Qwen2.5:1.5B 本地推理 + 预置提醒模板

#### 3.2.3 非现场执法模块
- **在线监测**：对接企业在线监测数据（水/气/噪声）
- **用电监控**：分析企业用电异常（停产时用电→偷排嫌疑）
- **视频监控**：AI 视频分析（烟尘/污水/扬尘识别）
- **无人机巡查**：定期航线巡查 + AI 图像分析
- **预警模型**：多源数据融合 → 风险评分 → 自动预警

#### 3.2.4 第三方监管督查模块
**监管人员穿戴设备流程**：
```
监管人员到达企业
  ↓
穿戴设备启动 → 显示督查清单
  ↓
AI 语音引导：「本次督查重点：
  1. 检查排污许可证有效性
  2. 核实在线监测设备运行
  3. 查看危废贮存间
  4. 审查环境管理台账」
  ↓
逐项检查 → 拍照取证 → 语音备注
  ↓
AI 自动生成督查报告
  ↓
发现问题 → 自动关联法条 → 建议处理措施
```

#### 3.2.5 群众举报模块
- **多渠道接入**：微信小程序 / 12369热线 / 网页 / APP
- **智能分发**：AI 分析举报内容 → 自动分类 → 分派至对应科室
- **流程跟踪**：举报→受理→核实→处置→公示→评价
- **激励机制**：有效举报积分奖励、实名举报优先处理

#### 3.2.6 闭环协同
- **数据流**：各模块数据统一汇聚至知识库
- **业务流**：线索→立案→检查→证据→审查→处罚→执行→结案
- **反馈机制**：处置结果反馈至知识库 → 优化算法模型
- **学习机制**：案例积累 → AI 持续学习 → 提升匹配准确率

## 五、技术选型建议

### 5.1 穿戴设备集成
- **通信协议**：蓝牙 5.0（传感器）+ WiFi/4G（记录仪/头盔）
- **数据格式**：JSON over WebSocket（实时）+ MQTT（低功耗传感器）
- **SDK**：各设备厂商 SDK + 自定义适配层

### 5.2 语音系统
- **ASR**：讯飞 ASR（中文最优）或 Whisper-Large（开源离线）
- **TTS**：CosyVoice（阿里开源，声音自然）或讯飞 TTS
- **离线方案**：Qwen2.5:1.5B + 预置提醒模板（无网络环境）

### 5.3 AI 分析
- **场景识别**：GPT-4o Vision（云端）+ YOLOv8（本地 60fps）
- **法条匹配**：LawMatch 算法（已实现）+ 向量检索（Milvus/Faiss）
- **文书生成**：LLM + 模板引擎（DocGen 规划中）

### 5.4 数据存储
- **当前**：JSON 文件 → **目标**：SQLite（轻量）+ Redis（缓存）
- **文件存储**：本地 → **目标**：MinIO/S3（可扩展）
- **时序数据**：传感器数据 → InfluxDB（高频写入）

## 七、专业补充（架构师视角）

### 7.1 数据安全
- **执法数据加密**：证据文件 AES-256 加密存储
- **传输安全**：穿戴设备 ↔ 服务器全程 TLS 1.3
- **访问控制**：RBAC 角色权限（执法/监管/行政/管理员）
- **审计日志**：所有操作可追溯，不可删除
- **数据脱敏**：举报人信息脱敏存储

### 7.2 离线能力（关键）
- 执法现场可能无网络 → 本地缓存 + 离线 AI
- Qwen2.5:1.5B 部署在执法终端设备上
- 语音提醒预置模板 + 本地 TTS
- 证据数据本地暂存 → 网络恢复后自动同步

### 7.3 合规要求
- 电子证据法律效力：时间戳 + 数字签名 + 区块链存证
- 执法全过程记录：符合《行政处罚法》要求
- 数据保留期限：案件档案 ≥ 10 年
- 个人信息保护：符合《个人信息保护法》

### 7.4 设备管理
- 设备注册与绑定
- 电量监控与低电预警
- 固件 OTA 升级
- 设备故障自动告警

### 7.5 多租户设计
- 不同生态环境局独立部署或共享实例
- 数据隔离（schema 级或行级）
- 可配置的知识库和法规库

### 7.6 性能考量
- 语音提醒延迟 < 2 秒（端到端）
- 场景识别延迟 < 1 秒（本地 YOLOv8）
- 法条匹配延迟 < 500ms
- 并发执法终端 ≥ 100 台
