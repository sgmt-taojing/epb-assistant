Phase 1 — 工程化基建（第1-4周）
P0 · 4周 · 前端+后端+测试
1.1 前端构建工具链搭建
P0
引入 Vite 构建工具，将59个HTML页面的内联CSS/JS提取为独立模块。建立 src/ → dist/ 构建流程。配置开发服务器热更新。不引入框架（保持原生JS），但实现模块化(import/export)和CSS变量统一管理。
交付: vite.config.js + src/目录结构 + dist/构建产物 + npm run dev / npm run build
1.2 后端架构升级：Flask + SQLite
P0
将 file_server.py 迁移为 Flask 应用。引入 SQLite 数据库替代静态JSON。创建数据模型（Enterprise/Device/Case/Law/Task/User）。将14个API端点迁移为 Flask Blueprint。保持向前兼容：GET /api/cases 返回相同JSON结构。
交付: app.py + models.py + routes/*.py + epb.db + 迁移脚本(import_json_to_db.py)
1.3 自动化测试框架搭建
P0
搭建 pytest 后端测试 + Playwright 前端E2E测试。后端覆盖14个API端点（状态码+响应结构+边界值）。前端覆盖核心流程：首页加载→侧栏菜单→页面跳转→设备商城→案件查询→诊断报告生成。目标覆盖率 ≥60%。
交付: tests/ 目录 + pytest.ini + playwright.config.js + 30+测试用例 + CI可运行
1.4 用户认证与鉴权系统
P0
基于 Flask-Login 实现 JWT 认证。5角色RBAC（公众/企业/执法/监管/政府）。login.html 对接真实登录API。API端点加鉴权中间件。前端 auth-guard.js 升级为真实token校验。密码 bcrypt 加密存储。
交付: auth.py + JWT中间件 + RBAC权限矩阵 + 登录/注册API + token刷新机制
2
Phase 2 — 核心功能强化（第5-10周）
P1 · 6周 · 业务功能深化
2.1 IoT设备接入引擎
P1
实现真实设备数据接入：HJ212协议解析器、MQTT消息订阅、Modbus寄存器读取。设备注册→心跳保活→数据采集→异常告警→自动断线重连。数据写入SQLite，支持时序查询。iot-diagnostic.html 对接真实数据生成诊断报告。
交付: protocols/hj212.py + protocols/mqtt_client.py + protocols/modbus.py + DeviceManager + 数据采集服务
2.2 案件全流程闭环
P1
实现案件 举报→立案→调查→审理→处罚→执行→归档 全流程。POST /api/report 写入数据库。案件状态机驱动流转。SLA超时自动告警。执法文书AI生成对接模板引擎。自由裁量计算器对接真实案例库。
交付: case_workflow.py + 状态机 + SLA引擎 + 文书模板5种 + 裁量基准表
2.3 国标对标与诊断报告引擎
P1
将6部国标从JSON硬编码迁移为数据库管理。诊断报告引擎：采集数据→匹配标准→计算占标率→生成诊断项→给出建议→输出PDF。支持自定义标准扩展。报告模板可配置。
交付: standards_engine.py + report_generator.py + PDF导出 + 自定义标准API
2.4 实时监测大屏与WebSocket
P1
引入 Flask-SocketIO 实现实时推送。ops-monitor.html 从轮询改为WebSocket订阅。传感器数据秒级推送。告警实时弹窗。在线用户数实时统计。大屏支持全屏模式。
交付: websocket_events.py + 前端Socket.IO客户端 + 实时大屏
2.5 数据扩充与种子数据
P1
企业档案从1条扩充到50+条（山东省真实行业分布）。设备从4台扩充到30+台（覆盖7类设备）。案件50条保持（已有）。法规28部保持。生成确定性模拟时序数据（30天×30设备×5分钟间隔=86400条）。
交付: seed_data.py + 50企业 + 30设备 + 86400条时序数据 + 数据生成脚本
3
Phase 3 — 优化与体验提升（第11-18周）
P2 · 8周 · 性能+体验+安全
3.1 响应式适配全覆盖
P2
45个缺@media的页面全部补齐1200px/768px双断点。移动端体验优化：触控目标≥44px、字体≥14px、表格横向滚动。PWA支持：manifest.json + Service Worker + 离线访问。
交付: 59页面响应式 + PWA配置 + Lighthouse移动端评分≥80
3.2 性能优化
P2
HTML/CSS/JS压缩（Vite构建）。图片WebP格式转换+懒加载。API响应Gzip。SQLite索引优化。前端路由按需加载（code splitting）。首屏加载 ≤2s（3G网络）。
交付: 构建产物 <500KB gzipped + Lighthouse性能评分≥90 + 性能基准测试报告
3.3 安全加固
P2
HTTPS强制（Let's Encrypt + Nginx反代）。CSP内容安全策略。XSS防护（DOMPurify）。CSRF Token。SQL注入防护（SQLAlchemy ORM）。API限流（Flask-Limiter）。安全头（HSTS/X-Frame-Options等）。
交付: Nginx配置 + 安全中间件 + OWASP Top 10扫描通过 + 渗透测试报告
3.4 AI助手升级：对接大模型API
P2
将前端规则引擎升级为后端大模型API（GLM/通义千问）。知识库RAG：法规向量检索+上下文注入。多轮对话记忆。5角色system prompt。流式响应（SSE）。意图识别+function calling。
交付: ai_engine.py + RAG管道 + 5角色prompt + SSE流式接口 + 对话历史
3.5 CI/CD流水线
P2
GitHub Actions：push→lint→test→build→deploy。自动部署到GitHub Pages（前端）+ Docker镜像（后端）。PR自动审查。版本号管理（语义化版本）。回滚机制。
交付: .github/workflows/ci.yml + Docker镜像 + 自动部署脚本 + 版本tag
4
Phase 4 — 生产部署与运营（第19-24周）
P3 · 6周 · 部署+监控+运营
4.1 Docker容器化部署
P2
完善Dockerfile + docker-compose.yml。Flask + Gunicorn + Nginx三层架构。SQLite→PostgreSQL迁移（可选）。数据卷持久化。健康检查。一键部署脚本 deploy.py 升级。
交付: docker-compose.prod.yml + Nginx.conf + Gunicorn配置 + 部署SOP文档
4.2 运营监控体系
P3
ops-monitor.html 对接真实指标：Prometheus采集 + Grafana仪表盘。业务指标（案件/设备/企业）+ 系统指标（CPU/内存/网络）+ API指标（QPS/延迟/错误率）。告警规则（钉钉/企业微信通知）。
交付: prometheus.yml + grafana/仪表盘JSON + 告警规则 + 通知渠道
4.3 文档体系
P3
API文档（OpenAPI/Swagger）。架构设计文档。部署SOP。用户手册（5角色）。开发规范。数据字典。运维手册。
交付: docs/ 目录 + Swagger UI + 7份文档 + 在线文档站
4.4 灾备与数据迁移
P3
数据库自动备份（每日增量+每周全量）。数据导出/导入工具。灾难恢复SOP（RTO ≤4h, RPO ≤24h）。数据迁移脚本（版本升级时使用）。
交付: backup.py + restore.py + 恢复SOP + RTO/RPO验证报告

⚠️ 风险与对策
风险概率影响对策
HJ212协议设备不兼容中高先用模拟器验证，预留协议适配层
大模型API成本超预算中中分级策略：规则引擎优先，复杂问题才调用大模型
SQLite并发瓶颈中中Phase 4可迁移PostgreSQL，WAL模式缓解
前端无框架导致维护成本高高中Phase 1模块化拆分+组件注入机制已缓解，必要时引入Vue/Lit
数据安全合规（环保执法数据）中高Phase 3安全加固+数据脱敏+审计日志
